1. Overview — what runs where

Machine Role
Dev PC Edit website code in Cursor. Optional local testing with npm run dev.
Tower PC Runs the API server, Frontier bot scripts, and Chrome. Must stay on during jobs.
Vercel Hosts the website only. Never runs zip checker or qualifier bots.
Tailscale Funnel Provides a public HTTPS URL so the live website can reach your tower PC.
Key idea The website sends job orders to the tower. The tower runs the bots and sends progress back. If either tower window is closed, the site shows Tower offline.

2. One-time setup on the tower PC

Do this once, or again after you update tower-server or bot files on your dev PC.

Step 2.1 — Confirm Google Drive folder layout

Files sync via Google Drive to the tower at G:\My Drive\:

G:\My Drive\ ├── bot\ │ ├── frontier_zipcheck_v2.py ← Zip checker (phase 1) │ └── frontier_checker60.py ← Qualifier (phase 2) └── tower-server\ ├── main.py ├── start.bat ├── requirements.txt ├── .env.example └── .env ← you create this

After editing on your dev PC, wait for Google Drive to finish syncing before starting the tower.

Step 2.2 — Install tower-server dependencies

Open Command Prompt and run one command at a time:

cd /d "G:\My Drive\tower-server" pip install -r requirements.txt

Step 2.3 — Create the .env file

cd /d "G:\My Drive\tower-server" copy .env.example .env

Open .env in Notepad and confirm these values:

TOWER_BOT_DIR=G:\My Drive\bot TOWER_PYTHON=python TOWER_JOBS_DIR=./jobs TOWER_PORT=8787
Important TOWER_PYTHON=python must point to the Python that has bot packages (pandas, selenium, undetected-chromedriver). Use system Python — not a tiny venv that only has FastAPI.

Step 2.4 — Install Tailscale

  1. Download Tailscale from https://tailscale.com
  2. Install and sign in (e.g. demalic@github)
  3. Leave Tailscale running in the background

Step 2.5 — Set the website environment variable on Vercel

You will get the exact URL in Part 3. The live site needs this variable:

Variable name Example value
VITE_TOWER_API_URL https://desktop-xxxxx.tail5a5189.ts.net

Rules: No trailing slash. Do not add /health at the end.

  1. Go to vercel.com → your yfm2 project
  2. Settings → Environment Variables
  3. Add or edit VITE_TOWER_API_URL
  4. Redeploy after changing the value

3. Starting the tower (every session)

You need TWO windows open Closing either window takes the tower offline. Keep both open for the entire time you want jobs to run.

Window 1 — Start the API server

  1. Open Command Prompt
  2. Run these commands (one at a time):
cd /d "G:\My Drive\tower-server" python main.py

Expected output:

Uvicorn running on http://0.0.0.0:8787

Leave this window open.

Quick test on the tower PC — open a browser and visit:

http://localhost:8787/health

You should see JSON with "ok": true and both scripts listed as found.

Window 2 — Expose the tower to the internet

  1. Open a second Command Prompt or PowerShell window
  2. Run:
tailscale funnel 8787

Tailscale prints an HTTPS URL, for example:

https://desktop-9r5uufo.tail5a5189.ts.net
  1. Copy that URL — you need it for Vercel
  2. Leave this window open
Normal behavior Visiting the bare funnel URL may show {"detail":"Not Found"}. That is expected. Use /health or /api/... paths instead.

Step 3.3 — Update Vercel with your funnel URL

  1. In Vercel, set VITE_TOWER_API_URL to your funnel URL from Window 2
  2. Redeploy if you changed the variable

Step 3.4 — Confirm on the live website

  1. Open https://yfm2-theta.vercel.app
  2. Log in as admin
  3. Go to the Eligibility tab
  4. Top right should show Tower online (green badge)

If it shows Tower offline (red): click the badge → dropdown shows help → click Check now after fixing Windows 1 and 2.

4. Daily checklist

Before running eligibility jobs:

  • Tower PC is on and logged in
  • Google Drive finished syncing (bot + tower-server folders)
  • Window 1 running: python main.py
  • Window 2 running: tailscale funnel 8787
  • Live site shows Tower online
  • Optional: http://localhost:8787/health returns ok

5. Running a test job from the website

  1. Eligibility tab → select Frontier
  2. Enter a 5-digit ZIP code
  3. Click Run Pipeline
  4. Watch Zip Checker (phase 1) then Qualifier (phase 2)
  5. Use Tower output at the bottom for live logs
If qualifier fails right after “Starting browser…” Chrome or ChromeDriver is the issue. Confirm Chrome is installed and frontier_checker60.py uses the correct version_main for your Chrome version.

If zip checker succeeded but qualifier failed, use Retry qualifier — it reuses the zip check CSV and does not download addresses again.

6. Troubleshooting

Problem Solution
Tower offline on website Restart Window 1 and Window 2. Confirm Vercel VITE_TOWER_API_URL matches current funnel URL.
cd fails to change drive Use cd /d "G:\My Drive\tower-server" — include /d and quotes.
PowerShell shows >> Unclosed quote — press Ctrl+C, open a new window, paste one command at a time.
Zip check OK, qualifier fails instantly Run qualifier manually on tower; fix Chrome / driver version.
“Tower URL not configured” Set VITE_TOWER_API_URL on Vercel and redeploy.
Funnel URL changed after reboot Run tailscale funnel 8787 again and update Vercel.

Manual health check (from any browser)

https://YOUR-FUNNEL-URL.ts.net/health

Manual qualifier test (on tower PC)

cd /d "G:\My Drive\bot" python frontier_checker60.py --csv "G:\My Drive\tower-server\jobs\JOB-ID\zipcheck_24153.csv" --no-vpn

Replace JOB-ID and ZIP with your real job folder from a previous run.
